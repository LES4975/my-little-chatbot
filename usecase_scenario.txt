시나리오 1: USB-C 전원을 통한 로봇 제어


1. 개요
사용자는 컴퓨터와 로봇(PD/QC 트리거 모듈)을 USB-C to USB-A 케이블로 연결해 전원을 공급한다.  로봇은 전원 스위치가 ON 상태일 때만 정상적으로 부팅 된다.  


 2. 관계
- Initiator: 사용자  
- Pre-condition:  
  - 로봇은 USB-C 포트를 통해 전원을 받을 수 있는 상태여야 한다.  
  - 컴퓨터는 USB 포트에 충분한 전력을 공급할 수 있어야 한다.  
- Post-condition:  
  - 사용자의 전원 스위치 조작에 따라 로봇이 부팅되거나 대기 상태로 전환된다.  




3. 기본 흐름
1. 사용자는 USB-A 단자를 컴퓨터에 연결하고, USB-C 단자를 로봇에 연결한다. 
2.  로봇의 전원 스위치가 ON 상태일 경우, 즉시 전력이 인가되고 로봇이 자동 부팅된다.  
3. 로봇의 전원 스위치가 OFF 상태라면 로봇은 꺼진 상태를 유지한다. 
4. 로봇의 전원 스위치를 OFF에서 ON으로 전환하면 로봇은 자동 부팅된다. 




4. 예외 흐름
E1. USB 전원이 약하거나 불안정한 경우  
	1. 트리거 모듈이 전압을 인가하지 못하고 로봇이 부팅되지 않을 수 있다.  
	2. 로봇에 전원 인가 실패 메시지가 표시되거나, LED 점등이 되지 않는다.  


E2. 전원 스위치 불량으로 인해 ON 상태가 감지되지 않을 경우  
	1. 스위치를 눌러도 내부 회로에 전원이 전달되지 않는다.  




5. 참고 사항
- 로봇의 동작 여부는 항상 전원 스위치 상태에 따라 결정된다.  
- 외부 전원 공급은 로봇을 대기 상태로 둘 수 있지만, 동작은 스위치에 의해 제어된다.

시나리오2: 로봇과의 대화


1. 개요
사용자가 로봇의 이름을 부르면 로봇이 대화 모드를 이행한다. 로봇의 응답은 LLM에 기반한다. 사용자와 로봇은 대화를 주고받는다. 사용자가 로봇에게 말을 걸지 않으면 로봇의 대화 모드가 종료된다.


2. Relationships
Initiator: 사용자
Pre-condition: 로봇은 상시로 마이크로부터 주변 소리를 입력받는다.
Post-condition: 로봇은 대화 모드를 종료하고 상시로 마이크로부터 주변 소리를 입력받으며 사용자가 부를 때까지 대기한다.


3. 기본 흐름
  1. 사용자가 로봇의 이름을 부른다.
  2. 로봇이 사용자의 부름을 입력받았음을 소리로 알린다.
  3. 사용자가 로봇에게 음성으로 말을 전달한다.
  4. 사용자의 음성을 입력한 로봇은 서버로부터 응답을 전달받아 스피커로 출력한다.
  5. 상기 1~4번의 흐름을 반복한다.
  6. 사용자가 일정 시간동안 로봇에게 말을 걸지 않으면 로봇은 대화 모드를 종료하고, 대화 모드가 종료되었다고 소리로 알린다.


4. 대안 흐름
  A1. 사용자가 직접 대화 모드를 종료하고자 하는 경우
            1. 사용자가 대화를 종료하고 싶다고 말한다.
            2. 로봇은 대화 모드를 종료하고, 대화 모드가 종료되었다고 소리로 알린다.










시나리오3: 사용자의 얼굴 추적
1. 개요
   로봇은 얼굴을 인식하면, 해당 정보를 내부 모듈을 통해 전송하고, 이에 따라 움직임을 제어 한다.
2. Relationships
- Initiator : 로봇
- Supportes : 사용자
- Pre-condition : 로봇은 사용자를 인식하고, 이에 따라 움직임을 제어한다.
- Post-condition : 로봇은 사용자를 지속적으로 트랙킹 한다.


3. 기본 흐름
1. 사용자가  로봇 앞에 선다.
2. 카메라가 얼굴을 인식 영역 내에서 감지한다.
3. 얼굴의 위치에 따라 로봇이 몸통을 돌리거나 고개를 움직인다.
4. 상기 1~3번의 흐름을 반복한다.
